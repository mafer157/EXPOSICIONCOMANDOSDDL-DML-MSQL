Claves ajenas

¿Que es?

Una clave ajena es un campo o conjunto de campos en una tabla que hace referencia a la

clave principal (o una clave candidata única) en otra tabla.

Proposito principal

El propósito principal de las claves ajenas es garantizar la integridad referencial de los datos.

Esto significa que las relaciones entre las tablas se mantienen consistentes y que no se permiten operaciones que podrían resultar en referencias a registros que no existen en la tabla relacionada.

Usos principales

1.- Establecen relaciones entre tablas.

2.- Mantienen la integridad referencial.

3.- Imponen restricciones de integridad.

4.- Facilitan la normalización de la base de datos.

5.- Ayudan en la optimización de consultas.

6.- Permiten la modificación en cascada.

Dominio D

¿Qué es?

En el análisis de la estructura relacional, se va a partir de los Escalares.

Un escalar es la menor unidad semántica de información, es decir, es un dato atómico.

La atomicidad se considera a nivel semántico, de modo que su descomposición produce una pérdida de significado.

Se define un Dominio, o Dominio Simple, como un conjunto de escalares.

¿Para qué sirve?

Integridad de los datos:

El dominio ayuda a garantizar la integridad de los datos al restringir los tipos de valores que se pueden almacenar en un campo determinado. Esto ayuda a prevenir la inserción de datos incorrectos o incoherentes en la base de datos.

¿Para qué sirve?

Consistencia de los datos:

Al definir un conjunto específico de valores permitidos para un campo, el dominio ayuda a mantener la consistencia de los datos en toda la base de datos. Esto asegura que los datos almacenados cumplan con ciertas reglas y estándares predefinidos.

¿Para qué sirve?

Validación de datos:

El dominio puede utilizarse para validar los datos antes de ser almacenados en la base de datos. Por ejemplo, si un campo tiene un dominio de valores enteros positivos, cualquier intento de ingresar un valor no válido (como un texto o un número negativo) será rechazado.

¿Para qué sirve?

Optimización de consultas:

Los dominios pueden ser utilizados en la creación de índices, lo que puede mejorar el rendimiento de las consultas al permitir búsquedas más rápidas y eficientes en la base de datos.

Características

-Cada atributo de una relación tiene asociado un Dominio Subyacente, de modo que el valor de las instancias del atributo se incluyen en él.
-No todos los valores del dominio subyacente deben aparecer como instancia del atributo.
-Diferentes atributos pueden tener asociado el mismo dominio subyacente.

Características

-Los dominios permiten eliminar errores del programador, restringiendo la comparación a atributos asociados al mismo dominio.
-Esto se cumple también sobre dominios que sean equivalentes aunque semánticamente diferentes.
-En SQL no se incluye el concepto de dominio, permitiendo comparación entre atributos equivalentes. 4 DOMIN

Ventajas

-Mejora el Rendimiento: Al limitar los tipos de datos y establecer restricciones, se puede mejorar el rendimiento de las consultas y operaciones en la base de datos.
-Facilita la Validación: Permite una validación más efectiva de los datos de entrada, reduciendo la posibilidad de errores y asegurando que solo datos válidos se almacenen en la base de datos.

Ventajas

-Integridad de Datos: Facilita la aplicación de restricciones que ayudan a mantener la integridad de los datos, evitando inconsistencias y errores.
-Consistencia de Datos: Un dominio garantiza que los datos almacenados en un campo cumplan con las reglas específicas, asegurando la coherencia y calidad de la información.

Desventajas

-Rigidez: Puede llevar a cierta rigidez, especialmente cuando es necesario cambiar las reglas o restricciones de un dominio existente, lo cual puede resultar complicado.
-Complejidad en el Diseño: La definición detallada de dominios puede aumentar la complejidad en el diseño de la base de datos, especialmente en sistemas grandes.
-Posible Pérdida de Flexibilidad: En algunos casos, la implementación estricta de dominios puede limitar la flexibilidad del sistema, especialmente si los requisitos cambian con el tiempo.

DOMINIOS POR INTENCIÓN

Se refieren a las diferentes categorías de datos que pueden ser almacenadas en una base de datos, dependiendo de la intención o propósito para el cual se utilizarán.

Dominio de Entidad

Este tipo de dominio representa entidades del mundo real, como personas, lugares u objetos.

Por ejemplo, en una base de datos de una biblioteca, el dominio de entidad podría incluir tablas para libros, autores y bibliotecas.

Dominio de Relación

El dominio representa relaciones entre diferentes entidades.

Por ejemplo, si en una base de datos se tiene una tabla de estudiantes y otra tabla de cursos, el dominio de relación podría ser la inscripción de estudiantes en cursos.

Dominio de Valor

Este tipo de dominio representa valores individuales dentro de una entidad o relación.

Por ejemplo, en una tabla de empleados, el dominio de valor para el atributo "nombre" serían los nombres de los empleados.

Dominio de Tiempo

En este caso, el dominio representa datos relacionados con el tiempo, como fechas, horas, períodos, etc.
Por ejemplo, en una base de datos de reservas de vuelos, el dominio de tiempo podría incluir la fecha y hora de salida de un vuelo.

Dominio de Control

Este tipo de dominio se refiere a datos relacionados con el control y la gestión de la base de datos misma, como permisos de usuario, registros de auditoría, etc.

Dominio de Proceso

Este tipo de dominio se refiere a datos que representan procesos o flujos de trabajo en una organización.

Por ejemplo, en una base de datos de gestión de proyectos, el dominio de proceso podría incluir datos sobre las tareas asignadas a diferentes miembros del equipo.

TIPOS DE DOMINIOS

POR SU EXTENSIÓN

Genéricos (gTLD):

Son extensiones de dominio genéricas que no están asociadas específicamente con un país o región.

Comúnmente utilizadas para una variedad de propósitos, como sitios web comerciales, organizaciones, redes, información, etc.

De nivel superior con código de país (ccTLD):

Son extensiones de dominio asociadas con un país o territorio específico.

Diseñadas para representar entidades geográficas y suelen reflejar el código de país ISO 3166-1 de la región.

Dominios de nivel superior patrocinados (sTLD):

Son extensiones de dominio respaldadas o patrocinadas por organizaciones específicas o grupos de interés.

Creados para representar una categoría particular de entidades, como entidades gubernamentales (.gov), instituciones educativas (.edu), o el ejército (.mil).

Dominios de nivel superior geográficos (geoTLD):

Son extensiones que están vinculadas geográficamente a una región específica.

Ejemplos incluyen .nyc para la ciudad de Nueva York, .london para Londres, .berlin para Berlín, etc.

Dominios de nivel superior de infraestructura (iTLD):

Son dominios reservados para infraestructuras de Internet.

El ejemplo más común es .arpa.

Dominios de nivel superior internacionalizados (IDN ccTLD y gTLD):

Permiten el uso de caracteres no ASCII en los nombres de dominio.

Ejemplos incluyen .рф para Rusia, .中国 para China, .भारत para India, etc.
